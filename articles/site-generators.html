Website design choices

The lifecycle of my personal site

HTML, CSS, Javascript

Last summer, I got a bit too much FOMO from all the enthusiasm for shiny front-end websites and decided to learn how to do it myself. I had learnt HTML and CSS at a short course whilst at university, and I knew a bit about back end development from previous work experience, so I was keen to delve a bit deeper and understand how websites work, from front to back.

On I went, going through the whole list of posts on Mozilla's Web Developer guide, from the very basics of HTML (although going a bit quickly through these posts), up until I got to React, Vue and Ember.

Once I got to the front-end frameworks, I figured I didn't actually have to learn all three of them, as I had already set my mind on using GatsbyJS. It's framework that uses React to generate static sites. I thought that would be a good use of my time, rather than learning React and using it to build a static site generator, I may as well use an existing framework that seems to have reliable documentation and online support.

React and GatsbyJS

And so it went. I learned quite a bit of the Gatsby processes, only to realise that once the websites got deployed, there was a weird zooming effect that happened with the Gatsby sites. As someone who had spent quite a lot of time trusting that things in Gatsby just worked magically, I was a bit frustrated. Anyway, I later came to realise that this is an issue that can be solved, but as mentioned above, I wasn't the most Javascript-savvy person and I was sure that there were many other alternatives to me sorting out some obscure zooming problem.

So, I trashed out that Gatsby project and attempted to borrow existing Gatsby templates (that is a thing). This worked surpringly well actually. I was using a template from Netlify, which was constantly deploying the website after a commit was pushed on Github. Pretty cool!

Because I got a bit obsessed with styling and ensure my website is responsive, I eventually stumbled onto a nasty error with Netlify not finding certain files needed to compile. This was a bit frustrating, and at this point I realised that I should understand React a bit better, before deploying a website with GatsbyJS.

When it came to learning React, a couple of other priorities got in the way and this just kept being pushed back in the pile of to-do's. In the meantime, I learnt about a few issues with indexing static site generators, along some other drawbacks on these fairly loaded programs. I took a pause from developing my site, since I was also getting demand to provide some help on the subject...

VueJS

On a funny twist of fate, a couple of months down the line, I got asked by a few students to help them with some web development coursework using Vue. I thought I would be able to help, from my long weeks crash coursing on how the DOM works, asynchronous Javascript, and other bits and bobs. So I looked up Vue and read about how it works. It's really actually quite simple!

So I learned it myself, and then I tutored the students who needed help to create a web app.

Full circle: Bare bones HTML and CSS, again

I recognise a lot of potential in static site generators, from their ease of use to their flexibility. If I was hoping to shift my career towards a full-stack role, I would've dedicated a couple of weeks to learn React and build a great site with either GatsbyJS or just bare bones React.

However, as much as I enjoy seeing website come to life, I'm not looking to get a job in the industry since it is looking very crowded and competitive, and it's not really my most favourite thing to work on.

Hence, I went back to basics with using just simple HTML, CSS and bootstrap. The main motivation that led me to discard static site generators, is that they're not necessarily that great in terms of accessibility, load times and indexing on search engines. I know there's ways around all three of those but as mentioned above, this isn't my full-time job or main hobby. I just want to document my work online, in a simple and hassle-free way.
